# 배포 환경변수 체크리스트 (Exam Builder)

## 필수 환경변수(공통)
- `AUTH_SECRET`
  - NextAuth 세션 서명 키
  - 생성: `openssl rand -base64 32` 또는 32자 이상 임의 문자열
- `AUTH_GOOGLE_ID`
- `AUTH_GOOGLE_SECRET`
  - Google OAuth Web client credentials
- `OPENAI_API_KEY`
  - OpenAI API 키
- `ADMIN_EMAILS`
  - 관리자 이메일 콤마 구분. 예: `admin1@example.com,admin2@example.com`
- `EXAM_SET_GENERATE_LIMIT`
  - exam set당 generate 버튼 최대 횟수(기본값 5)
- `CRON_SECRET`
  - 크론 배치 API 인증용 비밀값

## 런타임 모드별 DB 설정

### 로컬 개발
- 기본 동작은 `.env.local`에서 `SUPABASE_DB_URL` 미설정 시 자동으로 SQLite 사용
- `LOCAL_DB_PATH`는 `data/app.db` 권장
- 실제 권한/파일 경로 문제 없도록 `data/` 폴더가 쓰기 가능해야 함

### 배포(Supabase Postgres 사용 시)
- `SUPABASE_DB_URL` (Supabase Postgres connection string)
- Supabase Dashboard > Project Settings > Database 에서 connection string 확인
- `SUPABASE_URL` (https://[project-ref].supabase.co)
- `SUPABASE_SERVICE_ROLE_KEY` (private key, server only)
- `SUPABASE_STORAGE_BUCKET` (private bucket name, 예: `exam-source-images`)

## Google OAuth 설정(공통)
- OAuth 리디렉션 URI
  - 로컬: `http://localhost:3000/api/auth/callback/google`
  - 배포: `https://{your-domain}/api/auth/callback/google`
- 승인된 JS 오리진
  - 로컬: `http://localhost:3000`
  - 배포: `https://{your-domain}`

## Vercel 기준 입력값 매핑
- Vercel > Project > Settings > Environment Variables
  - `AUTH_SECRET` = 본문 환경변수 값
  - `AUTH_GOOGLE_ID` = Google OAuth Client ID
  - `AUTH_GOOGLE_SECRET` = Google OAuth Client Secret
  - `OPENAI_API_KEY` = OpenAI API key
  - `SUPABASE_DB_URL` = Supabase Postgres connection string
  - `SUPABASE_URL` = Supabase project URL
  - `SUPABASE_SERVICE_ROLE_KEY` = Supabase service role key
  - `SUPABASE_STORAGE_BUCKET` = private storage bucket
  - `ADMIN_EMAILS` = 관리자 이메일 목록
  - `EXAM_SET_GENERATE_LIMIT` = exam set generate 제한
  - `CRON_SECRET` = cron API 인증 비밀값
  - `LOCAL_DB_PATH` 는 배포 시 미설정 또는 빈 값
- 환경 구분: Production, Preview, Development 모두 필요 시 동일 값 설정

## 비밀값 관리 규칙
- `.env.local`/`.env`는 절대 커밋 X
- `git`에는 예시 템플릿만 (`.env.example`)만 유지
- 운영 비밀값은 플랫폼 Secret/Environment 변수로만 주입

## 기능별 빠른 점검 체크
- 홈에서 `Sign in with Google` 버튼 활성화 여부
- 로그인 후 `/dashboard` 접근 가능 여부
- 시험 생성 업로드 후 `Generate` 동작 여부
- 생성 후 `Save`로 draft 저장 확인
- draft `Publish` 후 `/exams/[id]` 풀이 가능 여부
- 미완료 시험에서 이탈 시 경고/이력 저장 동작
